# Twitch Mod Dashboard

Настольное приложение на Electron + React для модераторов Twitch‑каналов.  
Позволяет одновременно следить за несколькими чатами, быстро применять модерацию, смотреть логи сообщений и профили пользователей.

---

## Возможности

- Подключение к нескольким Twitch‑чатам одновременно (через IRC / tmi.js).
- Отображение чата с:
  - пометкой удалённых сообщений,
  - отображением очистки чата («Чат очищен модератором»),
  - локальной историей в рамках сессии.
- Панель модерации:
  - бан / разбан / таймауты (IRC + Helix),
  - удаление отдельных сообщений,
  - полная очистка чата (`/clear`).
- Лог сообщений по пользователю:
  - просмотр сообщений по всем каналам и по отдельному каналу,
  - бан / разбан / таймаут прямо из лога,
  - удаление конкретных сообщений из чата.
- Просмотр профиля пользователя:
  - данные из Twitch Helix `/users`,
  - аватар, баннер, описание, дата создания, количество просмотров и т.п.
- Список каналов:
  - собственный список каналов,
  - импорт «каналов, где вы модератор»,
  - импорт «каналов, на которые вы подписаны»,
  - фильтр «Все» / «Где я модератор»,
  - удаление отдельных каналов и очистка всего списка.
- Список зрителей:
  - список текущих зрителей для выбранного канала (через Helix + fallback по IRC),
  - подсветка модераторов и стримера,
  - контекстное меню для модерации и просмотра логов/профиля.

## Настройка Twitch OAuth

Приложение использует Helix‑API Twitch, поэтому нужен OAuth‑токен с нужными правами.

### Вариант 1. Прямой OAuth (рекомендуется)

1. Зайди на [https://dev.twitch.tv/console/apps](https://dev.twitch.tv/console/apps).
2. Создай новое приложение:
   - **Name:** любое.
   - **OAuth Redirect URL:**  
     `http://localhost:58585/auth/twitch/callback`
   - **Category:** Website или Application.
3. Сохрани `Client ID` и сгенерируй `Client Secret`.
4. Запусти приложение (см. раздел «Запуск в режиме разработки»).
5. В настройках приложения (в твоём интерфейсе) введи:
   - `Client ID`,
   - `Client Secret`.
6. Нажми кнопку **«Войти через Twitch»** (прямой логин):
   - браузер откроет страницу авторизации Twitch,
   - после успешного входа токены сохранятся в локальный `store` (`twitch.accessToken`, `twitch.refreshToken`, `twitch.scopes`, `twitch.userId`, `twitch.login`).

### Вариант 2. Авторизация через Dimandus

Если у тебя настроен режим `dimandus`:

1. В `store` / настройках выбери режим Dimandus.
2. Нажми кнопку **«Войти через Dimandus»**:
   - откроется страница Dimandus,
   - после завершения авторизации Dimandus вернёт токены в приложение.

В обоих случаях приложение запросит необходимые scope’ы:

```text
chat:read
chat:edit
moderation:read
moderator:manage:banned_users
moderator:manage:chat_messages
moderator:manage:chat_settings
moderator:manage:announcements
moderator:manage:shield_mode
moderator:read:shield_mode
moderator:read:chatters
user:read:moderated_channels
user:read:follows
user:write:chat
```

## Запуск в режиме разработки

> Имя скрипта может отличаться — смотри `package.json`. Ниже — самый типовой вариант.

1. Запусти dev‑сервер и Electron:

```bash
npm run dev
```

Обычно скрипт `dev`:

- поднимает Vite‑сервер на `http://localhost:5173` (renderer),
- стартует Electron (`main.js`), который в dev‑режиме открывает `http://localhost:5173` в `BrowserWindow`.

Если у тебя в `package.json` разделено на два скрипта (пример):

```json
"scripts": {
  "dev:renderer": "vite",
  "dev:main": "electron .",
  "dev": "concurrently \"npm run dev:renderer\" \"npm run dev:main\""
}
```

то запуск всё равно выглядит как:

```bash
npm run dev
```

После этого:

- появится окно Electron с дашбордом;
- в консоли DevTools можно увидеть логи `[App]`, `[TMI]`, `[Sidebar]` и т.д.

## Сборка production‑версии

Конкретные команды зависят от того, чем ты собираешь Electron (electron‑builder, electron‑forge, electron‑packager и т.д.). Типичный набор:

```json
"scripts": {
  "build:renderer": "vite build",
  "build:main": "tsc -p tsconfig.main.json",
  "build": "npm run build:renderer && npm run build:main",
  "dist": "electron-builder"
}
```

Тогда:

```bash
# Сборка renderer + main
npm run build

# Упаковка в установщик / дистрибутив
npm run dist
```

Посмотри в своём `package.json`, какие есть скрипты (`build`, `dist`, `electron:build` и т.п.), и подкорректируй этот раздел README.

## Типичный рабочий сценарий

1. Запустить приложение (`npm run dev`).
2. Залогиниться в Twitch через встроенную кнопку.
3. Добавить каналы:
   - вручную через «+»,
   - импортировать «Модерируемые» (`M`),
   - импортировать «Подписки» (`♥`).
4. Открыть чат канала (клик по каналу или через ПКМ → «Открыть чат»).
5. Следить за чатами:
   - ПКМ по сообщению → профиль / лог / модерация / таймаут / бан,
   - ПКМ по зрителю в списке → профиль / лог / модерация.
6. При необходимости:
   - открыть лог пользователя (лог и модерация по всем его сообщениям),
   - открыть профиль (подробная инфа из Helix).

## Известные ограничения

- Twitch официально **не предоставляет историю чата** через API, поэтому:
  - история сообщений видна только с момента запуска приложения,
  - либо из твоего локального лога (если ты её сам будешь сохранять).
- Некоторые операции требуют прав **модератора** на канале:
  - `getChannelChatters`,
  - `moderator:manage:*`,
  - `moderator:read:*`.
  Если токен/права не подходят, соответствующие запросы будут падать (в логе видно `Нет данных авторизации Twitch` / `Недостаточно прав доступа (Scopes)`).

## Полезно знать

- При ошибке `Login authentication failed` в TMI:
  - токен чата протух;
  - приложение пытается автоматически обновить его через `twitch:ensureAccessToken`;
  - если refresh неудачен — нужно **перелогиниться** в Twitch через интерфейс.
- Удаление/таймауты/баны обрабатываются через Helix, а визуально — через локальное состояние (`deleted`, `cleared`), чтобы UI синхронизировался с сервером.
